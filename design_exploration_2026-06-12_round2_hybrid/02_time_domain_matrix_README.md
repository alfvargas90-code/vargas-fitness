# Hybrid·02 — Time × Domain Matrix  (the true 2-D fusion)

**One-line POV:** A grid where time is the rows and domains are the columns — so the dashboard shows not just every metric, but *when each one is actually true*.

## The fusion
The deepest merge of 2 + 3: Pillars' three domain **columns** (Recovery / Activity / Body) crossed with Day Arc's chronological **rows** (Morning 06:40 → Midday·NOW → Evening → Night), hung on a left time-spine with the NOW row glowing. Each cell is that domain's reading at that phase.

The payoff is what the empty cells say. The matrix refuses to fake even cadence:
- **Recovery** is rich in the morning, then "holding / quiet" — because it *is* a morning truth.
- **Activity** is light at dawn, peaks at midday (the NOW cell), totals by evening.
- **Body** is a single 07:05 scale snapshot; midday/evening are honestly marked "no reading / once-daily."
- **Night** is forecast — tonight's 8h target, "rebuild charge," the moon anchoring the rest cell.

So the structure itself teaches how the data pipeline behaves.

## Best for
**Alfredo the System Architect.** This is the view that exposes the machine's rhythm — which signal updates when, what's measured vs forecast, where the quiet is intentional. No other layout makes the *temporal shape of the data* a first-class fact.

## Trade-off
Highest density and the steepest read — narrow cells, a lot to parse. It's a console you study, not a number you glance. Some cells are deliberately empty, which is honest but can read as "incomplete" to a casual eye until you get the concept.
